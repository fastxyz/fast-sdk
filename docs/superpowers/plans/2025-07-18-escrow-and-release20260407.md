# Escrow + Release20260407 + REST Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Escrow operations, Release20260407 transaction version, and migrate all proxy calls from JSON-RPC to REST API.

**Architecture:** Unified API — no V2 bifurcation. Escrow is added as a 13th variant to `makeOperation`. Release20260407 uses `claims: Array(Operation)` instead of `claim: ClaimType`. `TransactionBuilder.sign()` branches on version internally. All JSON-RPC proxy calls are replaced with REST HTTP calls; the JSON-RPC client is deleted entirely.

**Tech Stack:** TypeScript, Effect (Schema), `@mysten/bcs`, `vitest`

```typescript
import { Effect, Schema } from "effect";
import { JSONParse, JSONStringify } from "json-with-bigint";
import { parseRestError } from "./error";

const RestSuccessEnvelope = Schema.Struct({
  data: Schema.Unknown,
  meta: Schema.Struct({ timestamp: Schema.String }),
});

const RestErrorEnvelope = Schema.Struct({
  error: Schema.Struct({
    code: Schema.String,
    message: Schema.String,
    details: Schema.optional(Schema.Unknown),
  }),
  meta: Schema.Struct({ timestamp: Schema.String }),
});

/** HTTP timeout error for REST calls. */
export class RestTimeoutError {
  readonly _tag = "RestTimeoutError";
  constructor(
    readonly url: string,
    readonly timeoutMs: number,
  ) {}
}

/** Send a POST request to a REST endpoint. Returns the `data` field. */
export const restPost = (
  baseUrl: string,
  path: string,
  body: unknown,
  timeoutMs = 15_000,
) =>
  Effect.gen(function* () {
    const url = `${baseUrl}${path}`;
    const res = yield* Effect.tryPromise(() =>
      fetch(url, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSONStringify(body),
      }),
    );
    return yield* handleResponse(res);
  }).pipe(
    Effect.timeout(`${timeoutMs} millis`),
    Effect.catchTag("TimeoutException", () =>
      Effect.fail(new RestTimeoutError(path, timeoutMs)),
    ),
  );

/** Send a GET request to a REST endpoint. Returns the `data` field. */
export const restGet = (
  baseUrl: string,
  path: string,
  params?: Record<string, string | undefined>,
  timeoutMs = 15_000,
) =>
  Effect.gen(function* () {
    const query = params
      ? Object.entries(params)
          .filter(([, v]) => v !== undefined)
          .map(([k, v]) => `${encodeURIComponent(k)}=${encodeURIComponent(v!)}`)
          .join("&")
      : "";
    const url = query ? `${baseUrl}${path}?${query}` : `${baseUrl}${path}`;
    const res = yield* Effect.tryPromise(() => fetch(url));
    return yield* handleResponse(res);
  }).pipe(
    Effect.timeout(`${timeoutMs} millis`),
    Effect.catchTag("TimeoutException", () =>
      Effect.fail(new RestTimeoutError(path, timeoutMs)),
    ),
  );

/** Parse a fetch Response into either the success data or a typed error. */
const handleResponse = (res: globalThis.Response) =>
  Effect.gen(function* () {
    const text = yield* Effect.tryPromise(() => res.text());
    const json = JSONParse(text);
    if (!res.ok) {
      const errEnvelope = yield* Schema.decodeUnknown(RestErrorEnvelope)(json);
      return yield* parseRestError(errEnvelope.error);
    }
    const envelope = yield* Schema.decodeUnknown(RestSuccessEnvelope)(json);
    return envelope.data;
  });
```

```typescript
import {

  FaucetThresholdExceededError,
  FaucetThrottledError,

  IpRateLimitedError,

  readonly code: string;
  readonly message: string;
  readonly details?: unknown;

/** Parse a REST error envelope into a typed SDK error. */
export const parseRestError = (err: RestErrorBody): Effect.Effect<never, unknown> => {
  const { code, message, details } = err;
  const d = (details ?? {}) as Record<string, unknown>;

  switch (code) {
    case "INVALID_REQUEST":
      return Effect.fail(new InvalidRequestError({ message }));

    case "NOT_FOUND":
      return Effect.fail(new NotFoundError({ message }));

    case "FAUCET_AMOUNT_EXCEEDED":

    case "TOO_MANY_CERTIFICATES_REQUESTED":
      return Effect.fail(new TooManyCertificatesRequestedError({ message }));

    case "FAUCET_DISABLED":
      return Effect.fail(new FaucetDisabledError({ message }));

    case "UNEXPECTED_NONCE":
      return Effect.fail(
        new ProxyUnexpectedNonceError({
          message,
          txNonce: BigInt(d.tx_nonce as number),
          expectedNonce: BigInt(d.expected_nonce as number),
        }),
      );

    case "VERIFIER_SIGNATURES_INVALID":
      return Effect.fail(new VerifierSigsInvalidError({ message }));

    case "FAUCET_THROTTLED":
      return Effect.fail(new FaucetThrottledError({ message }));

    case "INTERNAL_ERROR":
      return Effect.fail(new GeneralError({ message }));

    case "UPSTREAM_ERROR":
      return Effect.fail(new UpstreamError({ message }));

    case "IP_RATE_LIMITED":
      return Effect.fail(
        new IpRateLimitedError({
          message,
          retryAfterSecs: (d.retry_after_secs as number) ?? 0,
        }),
      );

    case "SERVICE_UNAVAILABLE":
      return Effect.fail(new ServiceUnavailableError({ message }));

    default:
      return Effect.fail(new RestError({ status: 0, code, message, details }));
  }
```

## Important Notes

### Breaking Changes

1. **`ProviderOptions.rpcUrl` → `ProviderOptions.url`** — all users must update their provider construction.
2. **Default transaction version changed** from `Release20260319` to `Release20260407` — users relying on the default must either update or explicitly pass `version: "Release20260319"`.
3. **Removed error classes**: `JsonRpcProtocolError`, `RpcError`, `RpcTimeoutError`, `UnexpectedNonceError`, `InsufficientFundingError`, `InvalidSignatureError`, `PreviousTransactionPendingError`, `MissingEarlierConfirmationsError`, `CertificateTooYoungError`, `NonSubmittableOperationError`, `ValidatorGenericError`, `DatabaseError`, `FaucetTxnFailedError`. These are replaced by `RestError`, `RestTimeoutError`, `UpstreamError`, `NotFoundError`, `IpRateLimitedError`, `ServiceUnavailableError`.
4. **`getAccountInfo` no longer supports `certificateByNonce`** — use `getTransactionCertificates` instead.

### Encoding Details

- `EscrowConfigId` and `EscrowJobId` are 32-byte arrays — structurally identical to `TokenId`. In the palette system, they use the same schema (`TokenId` / `Uint8Array32`). In REST responses, they appear as hex strings.
- REST `submit-transaction` sends hex-encoded BCS bytes (not JSON-serialized serde). The SDK BCS-encodes the transaction and signature, then hex-encodes the bytes.
- REST `faucet-drip` accepts `recipient` as bech32m, `amount` as decimal string, `token_id` as hex string without `0x` prefix.
- All REST GET endpoints use path params (bech32m addresses) and query params (hex token IDs, numeric nonces/limits).
